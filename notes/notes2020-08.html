8月4号 周二

今日关键词：History 对象

History 对象的属性与方法

  • History 对象包含用户（在浏览器窗口中）访问过的 URL
  • History 对象是 window 对象的一部分，可通过 window.history 属性对其进行访问
  • length 属性：返回历史列表中的网址数
  • back() 方法：加载 history 列表中的前一个URL
  • forward() 方法：加载 history 列表中的后一个URL
  • go(num) 方法：加载 history 列表中的某个具体页面，num 是整数，正(负)数表示访问往前(后)第 num 个页面

正则表达式

  • 正则表达式(Regular Expression)：使用单个字符串来描述、匹配一系列符合某个句法规则的字符串搜索模式
  • 搜索模式可用于文本搜索和文本替换
  • 语法：/正则表达式主体/修饰符(可选)，例：/^[a-zA-Z0-9]$/g
  • 正则表达式修饰符
    修饰符 描述
    g 执行对大小写不敏感的匹配
    i 执行全局匹配（查找所有匹配而非在找到第一个匹配后停止）
    m 执行多行匹配
  • 正则表达式模式：
    方括号 "[ ]" 用于查找某个范围内的字符
    表达式 描述
    [abc] 查找方括号之间的任何字符
    [0-9] 查找任何从 0 至 9 的数字
    (x|y) 查找任何以 | 分隔的选项
    圆括号 "( )" 可以提取匹配的字符串
    表达式 描述
    (abc) 匹配字符串 abc，并返回到 $ 变量中(最多9个)，后面可以通过 $n 来访问第 n 个圆括号匹配到的内容
    (?:abc) 匹配字符串 abc，但不获取匹配结果，常与或字符(|)组合使用，如：(?:a|b) 匹配字符串 a 或者字符串 b
    abc(?=def)、abc(?!def) 正向肯定和否定预查，在任何匹配（不匹配）def 字符串开始处匹配查找字符串 abc，例如：a(?=b) 和 a(?!b) 分别匹配 ab 和 ac 中的a；
    (?<=def)abc、(?<!def)abc 反向肯定和否定预查，在任何匹配（不匹配）def 字符串开始处反向匹配查找字符串 abc，例如：(?<=b)a 和 (?<!b)a 分别匹配 ba 和 ca 中的a
    元字符是拥有特殊含义的字符
    元字符 描述
    \d,\D 匹配数字和非数字字符
    \s,\S 匹配不可见字符和可见字符，不可见字符有 \f,\n,\r,\t,\v
    \b,\D 匹配单词边界和非单词边界，“er\b” 和 "er\B" 分别可以匹配 never、verd 中的 er
    \w,\W \w 匹配包括下划线的单词字符，类似但不等价于[A-Za-z0-9_]；\W 匹配任何非单词字符，等价于[^A-Za-z0-9_]
    \num num 为一个正整数，，对所获取的匹配引用，如：(.)\1 匹配两个连续的相同字符
    \<,\> 匹配字符的开始与结束，如：\<is\> 匹配 it is a... 中的 is，但不匹配 this 中的 is
    \uxxxx 查找以十六进制数 xxxx 规定的 Unicode 字符
    \f,\n,\r,\t,\v 空字符，分别匹配换页符、换行符、回车符、制表符和垂直制表符
    量词：匹配字符串的个数
    量词 描述
    a+ 匹配至少一个 a 的字符串（>=1）
    a* 匹配零个或多个 a 的字符串（>=0）
    a? 匹配任零个或一个 a 的字符串（0 or 1）
    a{n} 匹配 n 个 a 的字符串（n）
    a{n,} 匹配大于等于 n 个 a 的字符串（>=n）
    a{n,m} 匹配 n 到 m 个 a 的字符串（[n,m]）
  • str.search(RE)：查找正则表达式的匹配内容（字串），并返回子串的起始位置
  • str.replace(RE,newStr)：用新的字符串 newStr 替换 str 中正则表达式 RE 匹配的内容
  • RE.test(str)：检测 str 中是否匹配到正则表达式 RE，返回布尔值
  • RE.exec(str)：检索字符串 str 中的正则表达式 RE 的匹配，并返回存放匹配结果的数组，如果没有匹配内容则返回 null

javaScript 模块化

  • 模块化的原因：项目很大时会有命名冲突、文件依赖等问题；
  • 模块化开发演变：
    1. 函数块：全局函数将代码包括在函数体内。缺点：全局变量污染和命名冲突；
    2. 命名空间：把函数和变量封装在对象里避免命名冲突。缺点：多层级嵌套，命名空间越来越长，暴露模块成员；
    3. 私有共有成员分离：将函数包裹在立即执行函数内，由于函数外部无法访问内部变量，可选择性对外暴露内部成员。
  • 创建模块原则：高内聚低耦合，模块内相关性高，模块间关联低
  • 模块使用场景：业务复杂、重复逻辑多、可拓展性要求高
  • CommonJS规范：
    1. 通过 require 方法来同步加载所要依赖的其他模块，再通过 exports 或者 module.exports 来导出需要暴露的接口；
    2. 其内部定义的变量是属于这个模块的，不会对外暴露，也就是说不会污染全局变量；
    3. CommonJS规范是同步加载模块的，因此适合服务器端但不适用于浏览器端。
  • AMD 规范：
    是非同步加载模块，允许函数回调，如Require.js，需提前加载所有模块；
    AMD 对于依赖的模块提前执行，提倡依赖前置。
  • CMD 规范：
    按需加载，不需要再模块开始就加载所有依赖；
    CMD 对于依赖的模块延迟执行，提倡依赖就近，只需要在用到某个模块时再require。
  • ES6 模块化规范：
    使用 import 关键字引入模块，通过 export 关键字导出模块；
    ES6目前无法在浏览器中执行，但较之前面的模块化规范更规范、更强大，会逐渐代替第三方模块化规范。

元素显示优先级

  • 由 HTML 页面中出现的次序确定，但 select 元素除外
  • 由 CSS 的 z-index 属性设置，值越大显示次序越大
  • 帧元素>表单元素>非表单元素
  • 有窗口元素>无窗口元素，z-index 只能在同类元素之间起作用
  • 帧元素：frameset
  • 常见表单元素：textarea、select、input等
  • 常见非表单元素：a、div、p、span、table等
  • 有窗口元素：select、object、frames等
  • 无窗口元素：大部分 html 普通元素

算法时间复杂度

  • 算法的时间复杂度是一个函数，它定性描述该算法的运行时间,常用大写 O 符号表述，不包括这个函数的低阶项和首项系数
  • 循环次数最多原则：只需关注循环次数最多的那段代码即可
  • 加法原则：最大的两块代码时间复杂度相加即为总的时间复杂度
  • 乘法原则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积
  • 常量级时间复杂度：循环次数固定不变（常量num），记为 O(1)
  • 线性级时间复杂度：循环次数为num x n，记为 O(n)
  • 对数级时间复杂度：循环次数为num x logn，记为 O(logn)，如二分法、欧几里得算法、幂运算
